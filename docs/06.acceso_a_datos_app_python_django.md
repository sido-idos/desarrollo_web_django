# 1. Acceso a datos con Django
---
## Integración con bases de datos

Django soporta múltiples motores de bases de datos. La configuración se realiza en el archivo `settings.py` del proyecto.

## Comparativa de Motores de Bases de Datos
---

### Bases de datos relacionales SQL soportadas

1. **Oracle:** Robusto y escalable para aplicaciones empresariales críticas con altos requisitos de rendimiento.
2. **MySQL:** Fácil de usar, ideal para la mayoría de aplicaciones web.
3. **PostgreSQL:** Altamente extensible con soporte avanzado para datos complejos y JSON. Ideal para consultas avanzadas.

    ```sql
        DATABASES = {
            'default': {
                'ENGINE': 'django.db.backends.postgresql',
                'NAME': 'nombre_base_de_datos',
                'USER': 'tu_usuario',
                'PASSWORD': 'tu_contraseña',
                'HOST': 'localhost',
                'PORT': '5432',
            }
        }
    ```
4. **SQLite:** Motor ligero integrado en Python. Ideal para desarrollo, pruebas y prototipos.

### Bases de datos NoSQL

Django permite trabajar con MongoDB a través de extensiones. Para usar MongoDB con Django:

1. Instalar Django:
```bash
pip install django
```
2. Configura el archivo settings.py:
```sql
    DATABASES = {
        'default': {
            'ENGINE': 'djongo',
            'NAME': 'nombre_base_de_datos',
        }
    }
```

### Paquetes necesarios para cada motor

Cada motor requiere paquetes adicionales que actúan como puentes entre Django y la base de datos:

- **PostgreSQL:** `pip install psycopg2` (o `psycopg2-binary` para algunas plataformas)
- **MySQL:** `pip install mysqlclient`
- **SQLite:** Incluido nativamente en Python
- **Oracle:** `pip install cx_Oracle`

## Django como ORM
---

El ORM (Object-Relational Mapping) permite interactuar con la BD usando objetos Python en lugar de SQL directo. Los "modelos" definen la estructura de las tablas y generan métodos automáticos para operaciones.

**Ventajas:**
- Abstracción: Cambiar de motor de BD sin modificar código
- Seguridad: Protección contra inyecciones SQL
- Simplicidad: Modelos como clases Python
- Optimización: Lazy evaluation, `select_related()` y `prefetch_related()` para mejorar rendimiento

Ejemplo de definición de un modelo en Django:

```python
from django.db import models

class Producto(models.Model):
    nombre = models.CharField(max_length=100)
    precio = models.DecimalField(max_digits=8, decimal_places=2)
    stock = models.IntegerField()
    
    def __str__(self):
        return self.nombre
```

## Migraciones en Django
---

Las migraciones documentan y aplican cambios en la estructura de la BD de forma controlada. Cada cambio genera un archivo que se puede ejecutar o revertir.

**Pasos clave:**
1. Ejecutar `python manage.py makemigrations` para detectar cambios y generar archivos
2. Ejecutar `python manage.py migrate` para aplicar los cambios a la BD
3. (Opcional) Revertir con `python manage.py migrate <app_name> <migration_number>`


## Sintaxis de Consultas en el ORM de Django
---

El ORM permite construir consultas de forma encadenada y segura:
- **Lazy Evaluation:** Las consultas se ejecutan solo cuando se necesitan
- **Filtros encadenados:** Combinar múltiples condiciones
- **Métodos avanzados:** `update()`, `delete()`, `select_related()`, `prefetch_related()`
    - Obtener todos los registros:
    ```python
    productos = Producto.objects.all()
    ```
    - Filtrar registros (por ejemplo, productos con precio
    mayor a 100):
    ```python
    productos_caros = Producto.objects.filter(precio__gt=100)
    ```
    - Ordenar resultados:
    ```python 
    productos_ordenados = Producto.objects.order_by('nombre')
    ```
    - Consultas complejas con Q objects:
    ```python
    from django.db.models import Q
    
    productos_filtrados = Producto.objects.filter(Q(precio__gt=50) & Q(stock__gte=10))
    ```
## Ejecución de Queries SQL en Django
---

Django ofrece dos métodos para SQL personalizado:

1. **`raw()`**: Ejecuta SQL y retorna instancias del modelo
```python
for producto in Producto.objects.raw(
    'SELECT * FROM miapp_producto WHERE precio > %s', [100]
    ):
print(producto.nombre)
```

2. **`connection.cursor()`**: Proporciona control total sobre la ejecución
```python
from django.db import connection

with connection.cursor() as cursor:
    cursor.execute("SELECT COUNT(*) FROM miapp_producto")
    row = cursor.fetchone()
    print("Número de productos:", row[0])
```

# 2. El ORM de Django
---
## Definición del Modelo y Conceptos del ORM

### ¿Qué es un Modelo?

Un modelo es la representación en código de una tabla en la BD. Se define como una clase que hereda de `models.Model` y cada atributo se traduce en una columna. Esto permite trabajar con objetos Python en lugar de SQL directo.

### Definición de Campos y Tipos de Datos

Cada atributo del modelo es un campo que determina el tipo de dato y sus restricciones.

**Campos comunes:**
- `CharField`: Texto corto (requiere `max_length`)
- `IntegerField`: Números enteros
- `DecimalField`: Números decimales (requiere `max_digits` y `decimal_places`)
- `DateField` / `DateTimeField`: Fechas y fechas con hora
- `EmailField`: Correos con validación integrada

Ejemplo de un modelo Cliente con campo de nombre, edad y email:
```python
from django.db import models

class Cliente(models.Model):
    nombre = models.CharField(max_length=100)
    edad = models.IntegerField()
    email = models.EmailField(unique=True)
    
    def __str__(self):
        return self.nombre
```

### Opciones de Campos

- `max_length`: Longitud máxima (CharField, TextField)
- `default`: Valor por defecto
- `null`: Permite valores nulos en la BD
- `blank`: Permite campos vacíos en formularios
- `unique`: Asegura valores únicos
- `verbose_name`: Nombre legible en la interfaz de admin

## Manejo de Claves Primarias
---

### Llaves Primarias Simples

Django crea automáticamente un campo `id` como llave primaria. Para usar un campo diferente, añade `primary_key=True`.

```python
class Producto(models.Model):
    codigo = models.CharField(max_length=10, primary_key=True)
    nombre = models.CharField(max_length=100)
    precio = models.DecimalField(max_digits=8, decimal_places=2)
    
    def __str__(self):
        return self.nombre
```

### Llaves Primarias Compuestas

Django no soporta claves compuestas de forma nativa. Se simula usando `unique_together` para garantizar que la combinación de dos o más campos sea única.

```python
class Inscripcion(models.Model):
    estudiante = models.ForeignKey('Estudiante', on_delete=models.CASCADE)
    curso = models.ForeignKey('Curso', on_delete=models.CASCADE)
    fecha_inscripcion = models.DateField()
    
    class Meta:
    unique_together = (('estudiante', 'curso'),)
```

## Operaciones CRUD en el Modelo
---

**Create (Crear):** Insertar registros
- Con `save()`: `cliente = Cliente(...); cliente.save()`
- Con `create()`: `Cliente.objects.create(nombre=...)`

**Read (Leer):** Obtener registros
- Todos: `Cliente.objects.all()`
- Uno: `Cliente.objects.get(email="juan@example.com")`
- Filtrar: `Cliente.objects.filter(edad__gt=25)`

**Update (Actualizar):** Modificar registros
- Obtener, modificar y guardar: `cliente = Cliente.objects.get(pk=1); cliente.edad = 31; cliente.save()`

**Delete (Borrar):** Eliminar registros
- `cliente = Cliente.objects.get(pk=1); cliente.delete()`

## Profundización y Buenas Prácticas
---

### Validación y Manejo de Errores

Captura excepciones durante operaciones CRUD. Por ejemplo, usa `DoesNotExist` al usar `get()`:

```python
from django.core.exceptions import ObjectDoesNotExist

try:
    cliente = Cliente.objects.get(email="noexiste@example.com")
except ObjectDoesNotExist:
    print("El cliente no existe.")
```

### Uso de Migraciones

Siempre que modifiques un modelo, ejecuta:
```bash
python manage.py makemigrations
python manage.py migrate
```
Esto asegura cambios controlados y revertibles.

### Optimización de Consultas

Usa `select_related()` para relaciones Muchos a Uno y `prefetch_related()` para Muchos a Muchos. Reducen las consultas a la BD y mejoran rendimiento.

# 3. Manejo de relaciones en el ORM de Django
---
## Relaciones: Muchos a Uno
---

Representa una jerarquía donde múltiples registros "hijo" están vinculados a un "padre". Se implementa con `ForeignKey`.

**Parámetro `on_delete`:**
- `CASCADE`: Borra todos los hijos si se borra el padre
- `PROTECT`: Impide borrar el padre si hay hijos
- `SET_NULL`: Establece el campo en nulo en los hijos
- `SET_DEFAULT`: Asigna un valor por defecto

Ejemplo: Supongamos que tenemos un sistema de gestión de inventario en el que cada producto pertenece a una categoría. La implementación sería la siguiente:

```python
from django.db import models

class Categoria(models.Model):
    nombre = models.CharField(max_length=100)
    descripcion = models.TextField(blank=True, null=True)
    def __str__(self):
        return self.nombre

class Producto(models.Model):
    nombre = models.CharField(max_length=100)
    precio = models.DecimalField(max_digits=8, decimal_places=2)
    descripcion = models.TextField(blank=True, null=True)
    categoria = models.ForeignKey(Categoria, on_delete=models.CASCADE)

    def __str__(self):
        return self.nombre
```

Análisis: Cada instancia de Producto se relaciona con una instancia de Categoria. Al usar `on_delete=models.CASCADE`, se garantiza que la eliminación de una categoría elimine todos los productos asociados, manteniendo la integridad de la base de datos.

### Optimización

Usa `select_related('campo')` para reducir consultas SQL en relaciones Muchos a Uno.

Valida que la entidad padre exista antes de asignarla.

## Relaciones Muchos a Muchos
---

Múltiples registros de un modelo se relacionan con múltiples registros de otro, sin jerarquía estricta. Se usan con `ManyToManyField`. Django crea automáticamente una tabla intermedia.

```python
from django.db import models

class Curso(models.Model):
    titulo = models.CharField(max_length=100)
    descripcion = models.TextField()

    def __str__(self):
        return self.titulo

class Estudiante(models.Model):
    nombre = models.CharField(max_length=100)
    email = models.EmailField(unique=True)
    cursos = models.ManyToManyField(Curso)

    def __str__(self):
        return self.nombre
```


### Relaciones con Entidad Intermedia

Cuando necesites datos adicionales sobre la relación, define un modelo intermedio y usa el parámetro `through`. Usa `unique_together` para evitar duplicados.

```python
from django.db import models

class Empleado(models.Model):
    nombre = models.CharField(max_length=100)

    def __str__(self):
        return self.nombre

class Proyecto(models.Model):
    titulo = models.CharField(max_length=100)
    descripcion = models.TextField()

    def __str__(self):
        return self.titulo

class Asignacion(models.Model):
    empleado = models.ForeignKey(Empleado, on_delete=models.CASCADE)
    proyecto = models.ForeignKey(Proyecto, on_delete=models.CASCADE)
    fecha_asignacion = models.DateField()
    rol = models.CharField(max_length=50)

    class Meta:
        unique_together = (('empleado','proyecto'),)

    def __str__(self):
        return f"{self.empleado.nombre} en {self.proyecto.titulo} como {self.rol}"
```

La opción `unique_together` garantiza que un empleado no se asigne más de una vez al mismo proyecto.
Se utilizan claves foráneas para establecer la relación con Empleado y Proyecto.

### Optimización y Consideraciones

- **Prefetch_related():** Cuando se recuperan datos de una relación Muchos a Muchos, se recomienda utilizar `prefetch_related()` para minimizar las consultas SQL y obtener de forma eficiente los datos relacionados.
- Es fundamental definir correctamente las restricciones de unicidad y los índices en la tabla
intermedia para mantener la integridad referencial y mejorar el rendimiento en consultas complejas.

## Relaciones Uno a Uno

Cada registro de un modelo está relacionado de manera exclusiva con un único registro de otro modelo. Este tipo de relación es útil para extender un modelo sin modificar su estructura original.

### Implementación en Django
Se implementa con el campo `OneToOneField`.

```python
from django.db import models
from django.contrib.auth.models import User

class Perfil(models.Model):
    usuario = models.OneToOneField(User, on_delete=models.CASCADE)
    direccion = models.CharField(max_length=255, blank=True, null=True)
    telefono = models.CharField(max_length=15, blank=True, null=True)
    fecha_nacimiento = models.DateField(blank=True, null=True)
    bio = models.TextField(blank=True)

    def __str__(self):
        return f"Perfil de {self.usuario.username}"
```

Detalles:
- Cada usuario del sistema tendrá un perfil único.
- La eliminación del usuario también eliminará su perfil por el uso de `on_delete=models.CASCADE`.

### Consideraciones y Buenas Prácticas

- Las relaciones Uno a Uno son ideales para extender modelos de terceros (por ejemplo, el modelo User de
Django) sin alterar su estructura interna.
- Se puede acceder de forma directa al modelo relacionado, lo que simplifica la lógica de negocio y la presentación de datos en la interfaz de usuario.

## Estrategias de Optimización y Mantenimiento
---

### Optimización de Consultas

- **select_related():** Útil para relaciones Muchos a Uno y Uno a Uno ya que reduce el número de consultas haciendo una única unión SQL.
- **prefetch_related():** Ideal para relaciones Muchos a Muchos ya que permite traer todos los datos relacionados en consultas separadas pero optimizadas.

### Índices y Restricciones

- Asegurar de que los campos utilizados en relaciones (especialmente en tablas intermedias) estén indexados mejora significativamente el rendimiento en búsquedas y filtrados.
- Restricciones como `unique_together` o `unique=True` garantizan la integridad de los datos y evitan la duplicidad en las relaciones.

### Manejo del Borrado

- El parámetro `on_delete` es crítico para definir cómo se comportan las relaciones ante la eliminación de datos.

### Documentación y Evolución del Esquema

- Mantener documentación actualizada del diseño del modelo y las relaciones implementadas facilita el mantenimiento, la colaboración y la adaptación de la base de datos a nuevas necesidades.
- Se recomienda utilizar herramientas de diagramación y comentarios en el código para describir la
función de cada relación.

# 4. Migraciones
---
## ¿Qué es una Migración?

Archivo que Django genera para documentar y aplicar cambios en la estructura de la BD. Contiene operaciones que transforman el esquema de una versión a otra.

### Funciones

- Sincroniza modelos Python con la estructura física de la BD
- Permite avanzar o retroceder en el historial de cambios
- Reduce errores manuales
- Facilita pruebas antes de producción
- Proporciona un registro audítable de cambios

### Problemas que Resuelve

- Automatiza cambios en la BD sin intervención manual
- Sincroniza equipos con control de versiones
- Permite revertir cambios problemáticos de forma segura
- Documenta la evolución del esquema

### Comandos Fundamentales

1. **`makemigrations`**: Detecta cambios en los modelos y genera archivos de migración
   - Ejecutar: `python manage.py makemigrations`
   - Genera archivo con operaciones necesarias

2. **`migrate`**: Aplica migraciones pendientes a la BD
   - Ejecutar: `python manage.py migrate`
   - Actualiza el registro interno de migraciones

### Migraciones en Equipos Colaborativos

Cuando clonas un repositorio o integras cambios, ejecuta `migrate` para sincronizar la BD con la última versión de los modelos.

**Caso Práctico 1:** Agregar un nuevo campo a un Modelo. Supongamos que tenemos un modelo básico para Cliente:

```python
from django.db import models

class Cliente(models.Model):
    nombre = models.CharField(max_length=100)
    email = models.EmailField(unique=True)
```

Decidimos agregar un nuevo campo, fecha_nacimiento, para almacenar la fecha de nacimiento del cliente:

```python
class Cliente(models.Model):
    nombre = models.CharField(max_length=100)
    email = models.EmailField(unique=True)
    fecha_nacimiento = models.DateField(null=True, blank=True)
```

Después de realizar el cambio, se ejecuta:

```bash
python manage.py makemigrations
```

El sistema genera un archivo de migración que contiene una operación como:

```python
migrations.AddField(
    model_name='cliente',
    name='fecha_nacimiento',
    field=models.DateField(null=True, blank=True),
),
```

Luego, al ejecutar:

```bash
python manage.py migrate
```

la columna fecha_nacimiento se añade a la tabla correspondiente en la base de datos.

**Caso Práctico 2:** Modificar el Tipo de Dato de un Campo, por ejemplo, cambiar un campo de CharField a TextField para permitir textos más largos. Tras modificar el modelo, `makemigrations` detectará el cambio
y generará una migración que, al aplicarse, ajustará el esquema de la base de datos sin perder la información existente. Si se detecta un problema tras aplicar una migración, se puede revertir a una
versión anterior utilizando:

```bash
python manage.py migrate <nombre_app> <número_de_migración>
```

## Buenas Prácticas en Migraciones
---

### Control de Versiones

- Sube todos los archivos de migración a Git
- Revisa archivos generados antes de aplicar, especialmente en producción

### Cambios Incrementales

- Realiza cambios pequeños y no acumulados
- Cada migración se ejecuta como transacción completa (todo se aplica o todo se revierte)

### Pruebas

- Prueba migraciones en desarrollo antes de producción
- Integra `makemigrations` y `migrate` en pipelines CI/CD

### Documentación

- Documenta cambios importantes y sus razones
- Mantén historial claro de migraciones

## Aspectos Avanzados en Migraciones
---

### Gestión de Dependencias entre Migraciones

- Cada archivo de migración puede depender de migraciones previas, y Django gestiona estas dependencias automáticamente. En proyectos complejos con múltiples aplicaciones, es importante entender el orden de ejecución y, en ocasiones, ajustar manualmente las dependencias en el archivo de migración para evitar conflictos.

### Personalización de Operaciones de Migración

- Django permite personalizar las migraciones mediante operaciones personalizadas, como por ejemplo, crear una operación que modifique datos antes de aplicar un cambio en el esquema.

### Migraciones Squashing

- Django ofrece la opción de *"squashing"* (fusionar) migraciones, que permite consolidar varias migraciones en un solo archivo, simplificando el mantenimiento y mejora el rendimiento durante la inicialización de la base de datos en nuevos entornos.

### Uso de Opciones Avanzadas en migrate

* **--plan:** Ejecuta `python manage.py migrate --plan` para ver el plan de migración sin aplicarlo.
* **--fake:** Permite marcar una migración como aplicada sin ejecutar realmente las operaciones. Esto puede ser útil cuando se corrige manualmente el esquema de la base de datos y se desea sincronizar el
estado interno de migraciones sin cambiar el contenido de la base de datos.

### Integración con Herramientas de Gestión de Bases de Datos

- Para grandes proyectos, es recomendable utilizar herramientas de gestión y visualización de bases de datos que permitan inspeccionar el estado del esquema antes y después de aplicar migraciones.

## Estrategias para el Mantenimiento a Largo Plazo
---

### Monitoreo y Auditoría del Esquema

- Realiza auditorías periódicas del esquema de la base de datos y revisa el historial de migraciones para asegurarte de que el sistema sigue siendo coherente y eficiente.

### Actualización Continua

- Mantente actualizado con las nuevas versiones y funcionalidades, y considera refactorizar las migraciones antiguas (por ejemplo, utilizando squashing) para aprovechar las últimas mejoras y optimizaciones.

# 5. Consultas personalizadas
---
## Consultas con el ORM de Django

### Recuperación Básica de Registros

El ORM permite acceder a datos sin SQL usando **QuerySets** con métodos como `all()`, `get()`, `filter()` y `exclude()`. Se pueden encadenar para consultas complejas. Ejemplos:

```python
# Obtener todos
clientes = Cliente.objects.all()
# Obtener uno
cliente = Cliente.objects.get(id=1)
# Filtrar
clientes_activos = Cliente.objects.filter(activo=True)
# Excluir y encadenar
clientes_filtrados = Cliente.objects.filter(activo=True).exclude(nombre__icontains='Test')
```

### Consultas Avanzadas

El ORM permite *lookups* avanzados: comparaciones, búsquedas de patrones, operaciones de fecha y más. Ordena resultados con `order_by()`:

```python
# Buscar y filtrar
clientes = Cliente.objects.filter(nombre__istartswith='A', edad__gt=25)
# Ordenar (- invierte el orden)
clientes_ordenados = Cliente.objects.filter(activo=True).order_by('apellido','-edad')
```

### Anotaciones para Campos Calculados

Las anotaciones añaden campos calculados (totales, promedios, conteos) directamente en la consulta:

```python
from django.db.models import Count, Avg

# Anotar con el número de pedidos
clientes_con_pedidos = Cliente.objects.annotate(total_pedidos=Count('pedido'))
# Calcular promedio de gastos
clientes_con_promedio = Cliente.objects.annotate(promedio_gasto=Avg('pedido__monto'))
```

### Exclusión y Selección de Campos

Para optimizar rendimiento o proteger información, excludes o selecciones campos específicos:

```python
# Excluir campo 'detalle'
clientes = Cliente.objects.defer('detalle')
# Seleccionar solo ciertos campos
clientes = Cliente.objects.only('nombre','email')
```

## Ejecución de Sentencias SQL Personalizadas
---

### Uso del Método raw()

`raw()` es una forma directa de ejecutar sentencias SQL y mapear los resultados a instancias de un modelo. Es ideal para consultas complejas que no se pueden expresar fácilmente con el ORM.

```python
consulta = "SELECT id, nombre, email FROM miapp_cliente WHERE edad > %s AND activo = %s"
clientes = Cliente.objects.raw(consulta, [30, True])

for cliente in clientes:
    print(cliente.nombre, cliente.email)
```

Al usar `raw()`, es crucial pasar los parámetros de forma segura para evitar inyecciones SQL, utilizando placeholders (%s) y una lista de parámetros.

### Uso de Conexiones y Cursores

Django permite trabajar directamente con conexiones y cursores a través del módulo `django.db.connection`. Esto es útil para ejecutar operaciones CRUD completas o procedimientos almacenados.

```python
from django.db import connection
with connection.cursor() as cursor:
    cursor.execute("SELECT COUNT(*) FROM miapp_cliente WHERE activo = %s",[True])
    resultado = cursor.fetchone()
    print("Número de clientes activos:", resultado[0])
```

Este enfoque es particularmente útil cuando se requiere manipular los resultados de una manera no estándar o cuando se ejecutan comandos SQL que no están soportados por el ORM.

### Invocación a Procedimientos Almacenados

En ciertos casos, puede ser necesario ejecutar procedimientos almacenados que realizan operaciones complejas en la base de datos. Utilizando cursores, es posible llamar a estos procedimientos y manejar sus resultados.

```python
with connection.cursor() as cursor:
    cursor.callproc('obtener_resumen_clientes', [True])
    resumen = cursor.fetchall()
    for fila in resumen:
        print("Resumen:", fila)
```

## Mapeo de Campos y Gestión de Resultados
---

### Mapeo Explícito de Campos en Consultas SQL

Cuando se utiliza `raw()` o se ejecuta SQL personalizado, es importante que los nombres de los campos en la consulta coincidan con los atributos del modelo o se definan mediante alias, lo que garantiza que Django pueda mapear correctamente los resultados a instancias del modelo.

```python
consulta =
    "SELECT id, nombre, email, fecha_registro AS registro FROM miapp_cliente WHERE activo = %s"
clientes = Cliente.objects.raw(consulta, [True])
```

Aquí, el alias registro debe ser reconocido o mapeado según la definición del modelo, para evitar discrepancias en el mapeo.

### Exclusión de Campos para Optimización

La exclusión de campos en consultas no solo protege datos sensibles, sino que también optimiza el rendimiento. El método `defer()` permite omitir campos que no son necesarios para una operación en particular.

```python
# Excluir el campo 'observaciones' para mejorar la velocidad de la consulta
clientes = Cliente.objects.defer('observaciones')
```

Esta práctica reduce la carga en la base de datos y el tiempo de respuesta en aplicaciones con grandes volúmenes de datos.

## Integración de Anotaciones y Parámetros en SQL Personalizado
---

### Combinación de Anotaciones con Filtros Avanzados

Las anotaciones no se limitan al ORM, sino que pueden ser combinadas con consultas SQL personalizadas para proporcionar resultados enriquecidos. Por ejemplo, puedes calcular totales o promedios y luego filtrar según esos valores.

```python
from django.db.models import Sum

# Anotar cada cliente con el total gastado y filtrar los que superen un umbral
clientes_con_total = Cliente.objects.annotate(total_gastado=Sum('pedido__monto')).filter(total_gastado__gt=1000)
```

### Pasando Parámetros a Consultas raw()

Es fundamental pasar parámetros a las consultas `raw()` de forma segura. Esto se logra utilizando *placeholders* y proporcionando una lista de valores que se insertan de forma segura en la consulta:

```python
consulta = "SELECT * FROM miapp_cliente WHERE edad > %s AND activo = %s"
clientes = Cliente.objects.raw(consulta, [35, True])

for cliente in clientes:
    print(cliente.nombre)
```

El uso correcto de parámetros previene inyecciones SQL y garantiza la integridad y seguridad de la aplicación.

## Operaciones CRUD mediante SQL Directo
---

Aunque el ORM es potente, existen escenarios donde se requiere realizar operaciones CRUD (Crear, Leer, Actualizar, Borrar) mediante SQL directo, ya sea por razones de rendimiento o por la complejidad de la operación.

### Crear Registros (INSERT)

```python
with connection.cursor() as cursor:
    cursor.execute(
    """ INSERT INTO miapp_cliente (nombre, email, activo, fecha_registro)
    VALUES (%s, %s, %s, NOW())""", ['María López', 'maria@example.com', True])
```

Esta operación inserta un nuevo registro en la tabla `miapp_cliente`, utilizando la función `NOW()` para registrar la fecha de inserción.

### Leer Registros (SELECT)

```python
with connection.cursor() as cursor:
    cursor.execute("SELECT id, nombre, email FROM miapp_cliente WHERE activo = %s", [True])
    registros = cursor.fetchall()
    for registro in registros:
        print("ID:", registro[0], "Nombre:", registro[1], "Email:", registro[2])
```

Esta consulta selecciona registros específicos y los muestra, permitiendo un procesamiento detallado de los datos recuperados.

### Actualizar Registros (UPDATE)

```python
with connection.cursor() as cursor:
    cursor.execute("UPDATE miapp_cliente SET activo = %s WHERE id = %s", [False, 5])
```

Este ejemplo actualiza el estado de un cliente, demostrando cómo realizar modificaciones directas mediante SQL.

### Eliminar Registros (DELETE)

```python
with connection.cursor() as cursor:
    cursor.execute("DELETE FROM miapp_cliente WHERE id = %s", [5])
```

La operación DELETE se utiliza para remover registros de la base de datos, asegurando que se cumplan las restricciones y relaciones definidas en el esquema.

## Invocación de Procedimiento Almacenados y Uso de Cursores
---

### Invocación a Procedimientos Almacenados

Los procedimientos almacenados permiten encapsular lógica compleja directamente en la base de datos. Con Django, es posible invocarlos mediante cursores.

```python
with connection.cursor() as cursor:
    cursor.callproc('obtener_informes_clientes', [True])
    resultados = cursor.fetchall()
    for resultado in resultados:
        print("Informe:", resultado)
``` 

Este ejemplo muestra cómo llamar a un procedimiento almacenado que podría, por ejemplo, devolver un informe detallado de clientes activos.

### Gestión Avanzada de Cursores

El uso de cursores en Django no solo permite ejecutar consultas, sino también gestionar transacciones complejas. Es posible abrir y cerrar cursores, controlar commits y rollbacks, y manejar excepciones para garantizar la integridad de los datos en operaciones críticas.

## Estrategias de Optimización y Buenas Prácticas en Consultas Personalizadas
---

### Optimización de Consultas con select_related() y prefetch_related()

- **select_related():** Es ideal para relaciones de tipo Muchos a Uno y Uno a Uno, permitiendo que
Django realice una única consulta SQL con JOINs para recuperar datos relacionados.

```python
clientes = Cliente.objects.select_related('pais').filter(activo=True)
```

- **prefetch_related():** Se utiliza para relaciones Muchos a Muchos o cuando se requiere recuperar
conjuntos de datos relacionados en múltiples consultas optimizadas.

```python
clientes = Cliente.objects.prefetch_related('pedido_set').all()
```

### Seguridad y Prevención de Inyecciones SQL

- Siempre utiliza placeholders (%s) y pasa los parámetros como listas o tuplas en las consultas SQL. Esto evita que datos maliciosos sean inyectados en la consulta.
- Asegúrate de validar y sanitizar los datos de entrada antes de incluirlos en consultas, tanto al utilizar el ORM como en SQL directo.

- Documenta las consultas complejas y el propósito de cada operación SQL o consulta ORM avanzada.
-  Revisa y refactoriza las consultas para aprovechar nuevas funcionalidades o mejoras en el ORM de Django, manteniendo el código limpio y eficiente.

### Pruebas y Monitorización
- Implementa pruebas automatizadas para asegurarte de que las consultas devuelven los resultados esperados y se comportan correctamente ante cambios en el esquema.
- Utiliza herramientas y logs para monitorizar el rendimiento de las consultas, identificando cuellos de botella y optimizando según sea necesario.

## Interacción Entre Aplicaciones, Modelos y Vistas

### Definición del Modelo

En el contexto CRUD, el modelo representa la estructura de datos. Por ejemplo, se puede definir un modelo Cliente con campos como nombre, email, teléfono y dirección. La definición del modelo se realiza en clientes/models.py:

```python
from django.db import models

class Cliente(models.Model):
    nombre = models.CharField(max_length=100)
    email = models.EmailField(unique=True)
    telefono = models.CharField(max_length=20, blank=True, null=True)
    direccion = models.TextField(blank=True, null=True)
    fecha_registro = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.nombre
```

El ORM de Django se encarga de mapear este modelo a una tabla en la base de datos, con el comando:

```bash
python manage.py makemigrations
python manage.py migrate
```

Se crea y actualiza el esquema de la base de datos, integrando la nueva estructura.

### Creación de Vistas para Operaciones CRUD

Las vistas en Django gestionan la lógica de negocio y la interacción entre el usuario y la base de datos. Se pueden implementar vistas basadas en funciones o en clases. Por ejemplo, para un CRUD básico de clientes, se pueden definir vistas que:

- Muestren la lista de clientes (Read).
- Permitan agregar un nuevo cliente (Create).
- Permitan editar la información de un cliente existente (Update).
- Permitan eliminar un cliente (Delete).

Un ejemplo de vista basada en clases utilizando generic views sería:

```python
from django.urls import reverse_lazy
from django.views.generic import ListView, CreateView, UpdateView, DeleteView
from .models import Cliente

class ClienteListView(ListView):
    model = Cliente
    template_name = 'clientes/cliente_list.html'

class ClienteCreateView(CreateView):
    model = Cliente
    fields = ['nombre','email','telefono','direccion']
    template_name = 'clientes/cliente_form.html'
    success_url = reverse_lazy('cliente_list')

class ClienteUpdateView(UpdateView):
    model = Cliente
    fields = ['nombre','email','telefono','direccion']
    template_name = 'clientes/cliente_form.html'
    success_url = reverse_lazy('cliente_list')

class ClienteDeleteView(DeleteView):
    model = Cliente
    template_name = 'clientes/cliente_confirm_delete.html'
    success_url = reverse_lazy('cliente_list')
```

Estas vistas aprovechan las abstracciones que ofrece Django para realizar operaciones CRUD sin necesidad de escribir código repetitivo.

### Formularios y Seguridad CSRF

Django ofrece protección CSRF automática. Incluye el token en todos los formularios POST:

```html
<form method="post">
    {% csrf_token %}
    {{ form.as_p }}
    <button type="submit">Guardar</button>
</form>
```

## Enrutamiento y Parámetros

Configurado en **urls.py**. Usa parámetros dinámicos como `<int:pk>` para operaciones CRUD:

```python
from django.urls import path
from .views import ClienteListView, ClienteCreateView, ClienteUpdateView, ClienteDeleteView

urlpatterns = [
    path('', ClienteListView.as_view(), name='cliente_list'),
    path('nuevo/', ClienteCreateView.as_view(), name='cliente_create'),
    path('editar/<int:pk>/', ClienteUpdateView.as_view(), name='cliente_update'),
    path('eliminar/<int:pk>/', ClienteDeleteView.as_view(), name='cliente_delete'),
]
```

## Operaciones CRUD mediante el ORM

### Creación de Registros

El ORM de Django simplifica la creación de registros en la base de datos. Esto se puede realizar a través de formularios o mediante la utilización del método `create()`.

```python
# Creación de un nuevo cliente mediante el método create()
nuevo_cliente = Cliente.objects.create(
                nombre='Juan Martínez',
                email='juan.martinez@example.com',
                telefono='123456789',
                direccion='Calle Falsa 123'
                )
```

### Lectura de Registros

La lectura de registros se realiza mediante métodos como `all()`, `filter()` o `get()`, permitiendo recuperar datos de forma flexible y con condiciones específicas.

```python
# Obtener todos los clientes activos
clientes_activos = Cliente.objects.filter(activo=True)
```

### Actualización de Registros

La actualización implica recuperar un registro, modificar sus atributos y guardar los cambios. El ORM facilita este proceso a través del método `save()`.

```python
cliente = Cliente.objects.get(pk=1)
cliente.telefono = '987654321'
cliente.save()
```

### Eliminación de Registros

La eliminación de registros se efectúa mediante el método `delete()`, que borra de forma permanente el registro de la base de datos.

```python
cliente = Cliente.objects.get(pk=1)
cliente.delete()
```

# 7. Aplicaciones Django preinstaladas

## Visión General de las Aplicaciones Preinstaladas

1. django.contrib.admin
    - Proporciona una interfaz administrativa lista para usar, que permite a los desarrolladores y administradores gestionar los modelos registrados sin necesidad de crear una interfaz desde cero.
    - Características Destacadas:
        - Interfaz de usuario intuitiva y personalizable.
        - Soporte para operaciones CRUD de manera automática.
        - Capacidades de búsqueda, filtrado, ordenamiento y edición en línea.
        - Permite una gestión rápida y efectiva de la información, reduciendo el tiempo de desarrollo y facilitando la administración de datos en entornos de producción.

```python
from django.contrib import admin
from .models import Producto

class ProductoAdmin(admin.ModelAdmin):
    list_display = ('nombre','precio','categoria')
    search_fields = ('nombre',)
    list_filter = ('categoria',)

admin.site.register(Producto, ProductoAdmin)
```

2. django.contrib.auth
    - Proporciona un sistema de autenticación y autorización completo. Esto incluye la gestión de usuarios, la asignación de permisos, la creación de grupos y el manejo de procesos de inicio y cierre de sesión.
    - Facilita la implementación de mecanismos de seguridad robustos, permitiendo a los desarrolladores centrarse en la lógica específica de la aplicación sin tener que construir desde cero un sistema de autenticación.
    - Características Destacadas:
        - Modelos predefinidos como User, Group y Permission.
        - Formularios y vistas integradas para la autenticación y gestión de usuarios.
        - Facilidad para extender o reemplazar el modelo de usuario predeterminado según las necesidades del proyecto.

```python        
from django.contrib.auth.models import User

# Listar todos los usuarios registrados
for user in User.objects.all():
    print(user.username, user.email)
```

3. django.contrib.contenttypes
    - Esta aplicación permite que Django gestione información sobre todos los modelos instalados en un proyecto, facilitando la creación de relaciones genéricas.
    - Características Destacadas:
        - El modelo ContentType almacena información sobre cada modelo, permitiendo su identificación y manejo dinámico.
        - Es fundamental para funcionalidades avanzadas como la asociación genérica de objetos y la definición de permisos a nivel de modelo.
        - Permite una mayor flexibilidad en el manejo de relaciones entre modelos y potencia la capacidad de Django para trabajar con estructuras de datos dinámicas.

4. django.contrib.sessions
    - Administra la persistencia de datos de sesión entre peticiones HTTP, permitiendo almacenar información en el servidor para cada usuario.
    - Características Destacadas:
        - Almacenamiento de datos de sesión en la base de datos o en otros backends configurables.
        - Gestión automática de la expiración y limpieza de sesiones.
        - Permite implementar funcionalidades personalizadas como carritos de compra, formularios multi-paso o almacenamiento temporal de datos sin exponer información en el cliente.

Puedes acceder a la sesión en las vistas de Django de la siguiente manera:

```python
def mi_vista(request):
    num_visitas = request.session.get('num_visitas', 0)
    request.session['num_visitas'] = num_visitas + 1
    return HttpResponse(f"Has visitado esta página {num_visitas} veces.")
```

5. django.contrib.messages
    - Proporciona un sistema de mensajería que permite enviar mensajes temporales al usuario, como notificaciones de éxito, advertencias o errores.
    - Características Destacadas:
        - Integración directa en las vistas y plantillas, permitiendo mostrar mensajes con facilidad.
        - Múltiples niveles de mensajes (informativo, éxito, advertencia, error).
        - Mejora la experiencia del usuario proporcionando retroalimentación inmediata sobre las acciones realizadas, lo cual es fundamental en aplicaciones interactivas.

```python
from django.contrib import messages
from django.shortcuts import redirect

def mi_vista(request):
    messages.success(request,"¡Operación exitosa!")
    return redirect('home')
```

6. django.contrib.staticfiles
    - Facilita la gestión, recolección y distribución de archivos estáticos, como hojas de estilo, JavaScript e imágenes, en el desarrollo y despliegue de la aplicación.
    - Características Destacadas:
        - Comando `collectstatic` para reunir todos los archivos en un solo directorio.
        - Soporte para múltiples backends de almacenamiento de archivos.
    - Automatiza la gestión de recursos estáticos, asegurando que la aplicación cargue los archivos necesarios de forma eficiente y organizada en entornos de desarrollo y producción.

En el archivo `settings.py` puedes configurar:
```python
STATIC_URL = '/static/'
STATICFILES_DIRS = [os.path.join(BASE_DIR, "static")]
```
